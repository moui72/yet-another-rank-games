# Plan: Seed list_entries in the sharing e2e test helper

## Goal

Fix `e2e/sharing.spec.ts`'s `seedPoolAndList` helper so it seeds `list_entries` rows, so tests asserting on rendered game names in a shared list have real data to find instead of hitting the empty "No games ranked yet" state.

## Scope

**In scope:**
- Update `seedPoolAndList` in `e2e/sharing.spec.ts` to insert `list_entries` rows (one per seeded game, with a `position` and `score`) alongside the existing `games`/`pools`/`pool_games`/`lists` inserts, addressing feedback F001.

**Out of scope:**
- The duplicate `seedPoolAndList` helper in `e2e/infoPopover.spec.ts` — a separate, independently-defined function; feedback F001 named only the sharing spec's copy as the one blocking a real test, so this plan doesn't touch the other file to avoid scope creep into unrelated tests that aren't failing on this gap.
- Any product/data-model change — `list_entries` is an existing table (see `datamodel.md`'s `ListEntry` entity); this is a test-fixture fix only.

## Technical Approach

`list_entries` (see `datamodel.md`'s `ListEntry` entity) is the derived-snapshot table the public share view reads to render game names. `seedPoolAndList` already inserts two games, a pool, `pool_games` rows, and a list — the fix inserts one `list_entries` row per seeded game (`list_id`, `game_id`, a `position`, e.g. 1 and 2 in insertion order) in the same transaction-adjacent block, before the function returns, so any test calling the helper gets a list whose shared view has real ranked games to assert against.
