# Plan: Fix e2e hydration-race flake in public share-view test

## Goal

Fix the pre-existing hydration-race flake in the "public /share/[token] view" e2e test so it reliably exercises the share flow instead of silently failing when its checkbox click races Svelte's client hydration.

## Scope

**In scope:**
- Make the `public /share/[token] view` test in `e2e/sharing.spec.ts` wait for client hydration before interacting with the "Share a read-only link" checkbox, addressing feedback F001.

**Out of scope:**
- Any other e2e spec that might share this hydration-race pattern — this plan fixes the one confirmed-affected test; if the same race turns up elsewhere, that's new feedback, not a scope expansion here.
- Any change to the sharing feature's actual behavior — this is a test-reliability fix only.

## Technical Approach

The `share toggle and copy-link` test in the same file was already hardened against this exact race with `page.waitForLoadState('networkidle')` before interacting with the page (landed as part of the clipboard-permissions fix in `plan-80b9-2026-07-24-fac9.md`). Apply the same fix here: add `await page.waitForLoadState('networkidle')` after `page.goto(`/lists/${listId}`)` and before checking the "Share a read-only link" checkbox, so the interaction can't fire before Svelte finishes attaching its event listeners.
